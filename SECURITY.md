# Security Policy 🛡️

## Supported Versions

Since SecureX is currently in the **Architecture & Systems Design** phase, we are currently threat modeling the foundational layers. Active software patches are not yet being deployed.

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: (Architectural Feedback only) |

## Reporting a Vulnerability

As a cybersecurity platform, we hold our design and future implementation to the highest security standards. If you discover a flaw in our architectural design or a potential exploit path, please proceed as follows:

1. **Do not open a public GitHub issue.**
2. Send a detailed architectural report to `security@securex.internal` (placeholder for internal testing / security audits).
3. Include:
   - Affected system components or files in `docs/`
   - Detailed threat vector or exploit scenarios (e.g., race conditions, token spoofing)
   - Proposed mitigation strategy or design modifications

We will respond within 48 hours to coordinate a responsible disclosure and update our threat model documents.
